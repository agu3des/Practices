Questão 1

As chamadas Redes de computadores são sistemas computacionais que interligam-se para o compartilhamento de dados. Os dispositivos conectados podem se diferenciar ou assemelhar em métricas de software e hardware.

Questão 2

A diferença mais notável no meu dia a dia foi a presença definitiva de tecnologias móveis, como o celular e  o notebook. Permitindo, por exemplo, uma comunicação imediata com minha mãe avisando que vou para casa.

Questão 3

A projeção seria de que: como ocorreu antigamente certas tecnologias vão passar de uso em laboratório para uso cotidiano das pessoas, a internet será muito mais rápida, e novos dispositivos de uso pessoal serão criados ou aperfeiçoados.

Questão 4

Sim, afinal a Internet é como uma rede de computadores de abrangência mundial. Pois, essa é um conjunto mundial de lans e wans conectadas, que são tipo de redes.

Questão 5

Os clientes são computadores que enviam solicitações para os servidores para recuperar informações. Os servidores são computadores que fornecem informações para dispositivos finais na rede. Assim, o cliente solicita e o servidor tem a função de prover o que foi solicitado.

Questão 6

As redes peer to peer conectam computadores semelhantes em uma base igual, sem medição de um terceiro. Estando geralmente próximos fisicamente, nesse sistema todos são considerados “clientes e servidores”. Um exemplo de aplicação das redes é a transferência de arquivos.

Questão 7

As redes peer to peer são fáceis de configurar, pois são menos complexas e possuem menor custo. No entanto, não há uma administração centralizada, o que não a torna tão segura, não são escaláveis e possuem um desempenho mais lento.

Questão 8

Redes não-estruturadas: não possuem uma estrutura definida na rede de sobreposição e não usam algoritmos específicos para organizar as conexões da rede. Cada nó estabelece conexões aleatoriamente entre outros nós. São fáceis de construir, permitem otimizações em diferentes regiões da sobreposição, altamente robustas devido às altas taxas de “rotatividade” dos pares na rede.

Redes estruturadas: usam algoritmos e critérios específicos para organizar as conexões da rede. Possuem protocolos que garantem que qualquer nó possa encaminhar uma busca a outro nó que disponha do arquivo desejado. O procedimento mais utilizado é usando o protocolo DHT (Distributed Hash Tables), ele organiza os processos através de uma tabela hash distribuída.

Questão 9

BitTorrent é um protocolo de transferência da Internet, dito isso, ele tem a função de baixar arquivos da Internet, o que permite com que, por exemplo, algumas pessoas consumam conteúdo pago, de graça por ele.

Questão 10

O blockchain é um mecanismo de banco de dados que permite o compartilhamento transparente de informações na rede de uma empresa. Dentro do blockchain, a rede peer to peer, que é uma série de computadores e servidores, atua como um nó. Ou seja, quando uma nova informação entra na rede, ela é propagada entre todos os nós da rede peer-to-peer, sendo essa informação encriptada e privada.

2. A codificação é o ato de converter uma mensagem em códigos. No qual, 4 fatores devem ser efetuados:
- A codificação entre hosts deve estar em formato adequado para o meio físico.
- As mensagens são convertidas primeiramente em bits pelo host emissor.
- Cada bit é codificado em um padrão de sons, de ondas de luz ou de impulsos elétricos, dependendo da mídia de rede
- O host destino recebe e decodifica os sinais para interpretar a mensagem.
1. A formatação é o conjunto de regras seguidas para uma mensagem ser enviada. Com isso, há um formato combinado para cartas e endereçamento de cartas necessário para a entrega adequada. Um quadro atua como um envelope fornecendo os endereços de destino e origem.
2. Método de Acesso: Os hosts em uma rede precisam saber quando começar a enviar mensagens e como proceder quando houver colisões.

Controle de fluxo: Os hosts de origem e destino usam o controle de fluxo para negociar a temporização correta e evitar sobrecarregar o destino, bem como certificar-se de que as informações tenham sido recebidas.

Limite de Tempo da Resposta: Os hosts na rede têm regras que especificam por quanto tempo aguardar respostas e que ação tomar se o limite de tempo da resposta for ultrapassado.

- A camada física define as características técnicas dos dispositivos elétricos e ópticos (fios, conectores, voltagens, taxa de dados, hubs e transceivers) do sistema.
- A camada de ligação de dados também é conhecida como camada de enlace ou link de dados. Esta camada detecta e, opcionalmente, corrige erros que possam acontecer no nível físico. É responsável pela transmissão e recepção (delimitação) de quadros e pelo controle de fluxo. Ela também estabelece um protocolo de comunicação entre sistemas diretamente conectados. Exemplo de protocolos nesta camada: PPP, LAPB (do X.25),NetBios.
- A camada de Rede é responsável pelo endereçamento dos pacotes, convertendo endereços lógicos (IP) em endereços físicos (MAC) , de forma que os pacotes consigam chegar corretamente ao destino. Essa camada também determina a rota que os pacotes irão seguir para atingir o destino, baseada em fatores como condições de tráfego da rede e prioridades.
- A camada de transporte é responsável por usar os dados enviados pela camada de Sessão e dividi-los em pacotes que serão transmitidos para a Camada de Rede. No receptor, a camada de Transporte é responsável por pegar os pacotes recebidos da camada de Rede, remontar o dado original e assim enviá-lo à camada de Sessão.
- A camada de sessão permite que duas aplicações em computadores diferentes estabeleçam uma sessão de comunicação. Nesta sessão, essas aplicações definem como será feita a transmissão de dados e coloca marcações nos dados que serão transmitidos. Se porventura a rede falhar, os computadores reiniciam a transmissão dos dados a partir da última marcação recebida pelo computador receptor.
- A camada de Apresentação, também chamada camada de Tradução, converte o formato do dado recebido pela camada de Aplicação em um formato comum a ser usado na transmissão desse dado, ou seja, um formato entendido pelo protocolo usado. Um exemplo comum é a conversão do padrão de caracteres (código de página) quando o dispositivo transmissor usa um padrão diferente do ASCII. Pode ter outros usos, como compressão de dados e criptografia.
- A camada de aplicação é responsável por identificar e estabelecer a aplicação (programa) o qual será utilizado entre a máquina destinatária e o usuário como também disponibiliza os recursos (protocolo) para que tal comunicação aconteça. Por exemplo, ao solicitar a recepção de e-mail através do aplicativo de e-mail, este entrará em contato com a camada de Aplicação do protocolo de rede efetuando tal solicitação(POP3, IMAP). Tudo nesta camada é direcionado aos aplicativos. Alguns protocolos utilizados nesta camada são: HTTP, SMTP, FTP, SSH, RTP, Telnet, SIP, RDP, IRC, SNMP, NNTP, POP3, IMAP. BitTorrent, DNS, Ping
1. O TCP/IP (Transmission Control Protocol/Internet Protocol) representa um conjunto de protocolos que permitem que diversos equipamentos que constituem uma rede possam se comunicar entre si. É um protocolo estruturado por camadas na qual cada camada utiliza e presta serviços às camadas adjacentes. Cada camada apenas trata das informações que correspondem à sua função. O modelo TCP/IP quando comparado com o modelo OSI, tem duas camadas que se formam a partir da fusão de algumas camadas do modelo OSI, elas são: as camadas de Aplicação (Aplicação, Apresentação e Sessão) e Acesso à Rede (Ligação de dados e Física).

Existem 5 camadas distintas que formam o TCP/IP:

- A camada de aplicação é onde são realizadas a maior parte das requisições para execução de tarefas na rede. Formada por: TELNET (Terminal Virtual); FTP (File Transfer Protocol); SMTP (Send Mail Transfer Protocol); DNS (Domain Name System); HTTP (Hypertext Transfer Protocol).
- A camada de transporte é onde são executadas ações relacionadas à confiabilidade e integridade dos dados por meio de funções como o controle de fluxo, controle de erro, sequenciação e multiplexação de mensagens. Formada por: UDP (User Datagram Protocol) e o TCP (Transmission Control Protocol).
- A camada de internet, também conhecida como inter-redes, é responsável pela permissão de envio de pacotes por hosts a qualquer rede e pela garantia de que esses dados cheguem ao seu destino final. Formada por: IP (Internet Protocol) e ICMP (Internet Control Message Protocol).
- A camada Interface de rede corresponde às camadas de enlace de dados e física do OSI, onde ocorre a conexão básica do host com a rede por meio de algum protocolo capaz de enviar pacotes IP. É por meio desta camada que é possível transmitir dados a outros computadores dentro de uma mesma rede física. Formada por: Ethernet para Redes Locais (LAN – Local Area Network) e PPP (Point-to-Point Protocol) para Redes de Longa Distância (WAN – Wide Area Network).

1. A conexão física pode ser uma conexão com fio usando um cabo ou uma conexão sem fio usando ondas de rádio.

2. É o hardware do adaptador de rede na forma de uma placa de expansão que se encaixa em um slot de expansão na placa-mãe de um computador. Conectam um dispositivo a uma rede.

3. Os objetivos da camada física são 3:

- Fornece os meios para transportar os bits que formam o quadro da camada de enlace de dados na mídia de rede.
- Ela aceita um quadro completo da camada de enlace de dados e o codifica como uma série de sinais que são transmitidos para o meio físico local.
- Os bits codificados que formam um quadro são recebidos por um dispositivo final ou por um dispositivo intermediário.

4. As três formas básicas de meio físico de rede:

- Sinais elétricos → cabo de cobre
- Pulso de luz → cabo de fibra óptica
- Sinais de microondas → sem fio

5. A codificação é o método de converter um fluxo de bits de dados em um "código" predefinido.

6. Método de sinalização:

- Método de representação de bits.
- Os padrões da camada física devem definir que tipo de sinal representa um "1" e que tipo de sinal representa um "0".
- Um pulso longo pode representar o valor 1, enquanto um pulso curto representa o valor 0.

7. A largura de banda é a capacidade de um meio transportar dados.

8. A taxa de transferência é a medida da transferência de bits através do meio físico durante um determinado período.

9. Dois dos principais problemas que podem ocorrer com o cabeamento de cobre são:

- Interferência eletromagnética (EMI) ou interferência de radiofrequência (RFI) → os sinais podem distorcer e corromper os sinais de dados transportados pela mídia de cobre.
- Crosstalk → é uma perturbação causada pelos campos elétricos ou magnéticos de um sinal em um fio para o sinal em um fio adjacente.

11. Os tipos de cabos de cobre existentes são:

- Cabo de par trançado não blindado → UTP
- Cabo de par trançado blindado → STP
- Cabo coaxial

12. Os itens que compõem a estrutura de um cabo UTP são:  revestimento externo, par trançado e isolamento em plástico com código de cores.

13. Os itens que compõem a estrutura de um cabo STP são: capa, proteção metálica ou trançada, proteção metálica e pares trançados.

14.  Os cabo STP diferem dos cabos UTP por serem cabos que possuem malha metálica (STP – Shielded Twisted Pair, em inglês), que podem envolver todo o cabo, chamada blindagem global  ou cada par que seria denominada blindagem dos pares.

15. Características do cabo Coaxial:

- Um condutor de cobre usado para transmitir os sinais elétricos.
- Uma camada de isolamento plástico flexível em volta de um condutor de cobre.
- O material de isolamento é envolvido em uma malha de cobre com tecido, ou uma folha metálica, que atua como o segundo cabo no circuito e uma proteção para o condutor interno.
- Todo o cabo é coberto com um revestimento para evitar danos físicos menores.

17. Os tipos de cabeamento UTP:

- Ethernet Direto → ambas as extremidades T568A o T568B, conecta um host a um dispositivo de rede, como um switch ou hub.
- Ethernet Cruzado → uma extremidade T568A e outra extremidade T568B, conecta dois hosts, conecta dois dispositivos de rede intermediários (switch com switch ou roteador com roteador).
- Rollover → proprietário da cisco, conecta uma porta serial da estação de trabalho à porta de console do roteador, usando um adaptador.

18. As propriedades do cabeamento de fibra óptica:

- Transmite dados por longas distâncias e em maior largura de banda.
- Transmite sinais com menos atenuação e é completamente imune à EMI e à RFI.
- Usado para interconectar dispositivos de rede.
- Fio flexível, extremamente fino e transparente de vidro muito puro, não muito maior do que um fio de cabelo humano.
- Os bits são codificados na fibra como pulsos de luz.

19. Componentes do cabo de fibra óptica:

- Capa → protege a fibra contra a abrasão, umidade e outros contaminantes. A composição pode variar dependendo do uso do cabo.
- Material de reforço → envolve o buffer, impedindo que o cabo de fibra seja esticado quando for puxado. Frequentemente, o mesmo material é usado para produzir coletes à prova de balas.
- Buffer → usado para ajudar a proteger o núcleo e o revestimento interno contra danos.
- Revestimento Interno → tende a atuar como espelho, refletindo a luz de volta ao núcleo da fibra. Mantém a luz no núcleo ao passar pela fibra.
- Núcleo → elemento de transmissão de luz no centro da fibra óptica. O núcleo normalmente é de silício ou vidro. Os pulsos de luz passam pelo núcleo da fibra.

20.  Tipos de cabos de fibra óptica:

- Multimodo → sigla MM, primeiro modelo no mercado, espessura maior (núcleo), luz dentro do cabo reflete mais (leds de luz), volume maior de perdas. Núcleo: vidro e 50/62.5 mícrons, Revestimento Interno: vidro e 124 mícrons de diâmetro, Revestimento.
- Monomodo → sigla SM, núcleo de espessura menor, menos perdas e distância maior, usado como backbones de campus, lasers como fonte de luz. Núcleo: vidro e 9 mícrons, Revestimento Interno: vidro e 125 mícrons de diâmetro, Revestimento polimérico.

21. Tipos de conectores de fibra óptica:

A luz só pode viajar em uma direção na fibra óptica, duas fibras são necessárias para a operação full duplex.

- Conectores de Ponta Reta (Straight-Tip – ST) → um dos primeiros tipos de conectores usados. Bloqueia com segurança ao "rosquear/desrosquear".
- Conectores SC (Conectores de Assinante) → chamados de conector quadrado ou padrão. Usam um mecanismo "push-pull" para assegurar a inserção correta. Usados com fibra de multimodo e monomodo.
- Conectores Lucent (LC) Simplex → versões menores do SC e populares devido ao

tamanho.

- Conectores LC duplex, multimodo → similares ao LC mas usam um conector duplex.

23. Aspectos a serem considerados sobre o meio físico sem fio:

- Área de cobertura → os materiais de construção utilizados em prédios e estruturas, e o terreno local, limitarão a cobertura.
- Interferência → prejudicada por dispositivos comuns, como luzes fluorescentes, fornos de micro-ondas e outras comunicações sem fio.
- Segurança → os dispositivos e usuários que não estão autorizados a acessar a rede podem obter acesso à transmissão.
- Meio compartilhado → apenas um dispositivo pode enviar ou receber de cada vez e o meio físico sem fio é compartilhado com todos os usuários sem fio.

1. Quais são os padrões que definem o protocolo Ethernet?

Definida nos padrões IEEE 802.2 e 802.3.

1. Em quais camadas do modelo OSI o protocolo Ethernet opera?

Ela opera na camada de enlace de dados e na camada física.

1. Quais as duas subcamadas da camada de enlace que o protocolo Ethernet utiliza?

As subcamadas LLC (controle de link lógico) e MAC.

1. **Quais as responsabilidades da camada MAC?**

Responsável pelo encapsulamento de dados e o controle de acesso ao meio.

1. Quais as funções principais do encapsulamento de dados da camada subcamada MAC?
- Delimitação de quadros
- Endereçamento
- Detecção de erros
1. O que são fragmentos de colisão e quadros jumbo?

Os quadros com comprimento menor que 64 são chamados de "fragmento de colisão" ou "quadro desprezível" e são automaticamente descartados pelas estações receptoras. Quadros com mais de 1.500 bytes de dados são considerados "jumbo" ou "baby giant".

1. O que é o endereço exclusivo da organização do endereço MAC?

O IEEE atribui ao fornecedor um código de 3 bytes (24 bits), na qual o IEEE exige que o fornecedor siga duas regras:

- Todos os endereços MAC atribuídos a uma NIC ou outro dispositivo Ethernet devem utilizar o OUI designado para o fornecedor como os primeiros 3 bytes.
- Todos os endereços MAC com o mesmo OUI devem receber um valor exclusivo nos últimos 3 bytes.
1. O que são endereços mac de unicast, multicast e broadcast?

Um endereço MAC unicast ****é o endereço exclusivo usado quando um quadro é enviado de um único dispositivo transmissor para um único dispositivo de destino.

Um pacote de broadcast contém um Endereço IPv4 de destino que tem todos os números 1 (1s) na porção de host, indicando que todos os hosts naquela rede local receberão e processarão o pacote.

Os endereços multicast permitem que um dispositivo de origem envie um pacote a um grupo de dispositivos.

1. **O que é a tabela de endereçamento mac do switch ?**

As tabelas de endereços MAC registram vários endereços MAC para as portas que conectam os switches que refletem além do nó. Um switch ligado terá uma tabela de endereço MAC vazia, pois ele ainda não aprendeu os endereços MAC dos quatro PCs conectados.

1. **Como o switch aprende os endereços MAC dos computadores conectados a ele?**

O switch cria dinamicamente a tabela de endereços MAC. Se o endereço MAC de origem for desconhecido, será adicionado à tabela juntamente com o número da porta. Se o endereço MAC de origem existe, o switch atualiza o timer de atualização dessa entrada.

1. **O que é e como funciona a filtragem de quadros que o switch executa?**

A medida que um switch recebe quadros de dispositivos diferentes, ele é capaz de preencher sua tabela de endereços MAC examinando o endereço MAC de origem de cada quadro. Quando a tabela de endereços MAC do switch contiver o endereço MAC de destino, ele poderá filtrar o quadro e encaminhar para uma única porta.

1. **Quais são os métodos de encaminhamento de quadros dos switches CISCO?**

Um switch store-and-forward recebe o quadro inteiro e calcula o CRC, se o CRC é válido, o switch procura o endereço de destino, que determina a interface de saída, o quadro é então encaminhado pela porta correta. Um switch cut-through encaminha o quadro antes que ele seja completamente recebido, pelo menos o endereço e destino do quadro deve ser lido para que o quadro possa ser encaminhado .

1. **Diferencie o método Store-and-forward do método Cut-Through?**

store-and-forward recebe o quadro inteiro e calcula o CRC, já um switch cut-through encaminha o quadro antes que ele seja completamente recebido.

1. **Quais são as formas de switching Cut-Through? Cite-as e diferencie.**

O switch fast-forward oferece o menor nível de latência, ele encaminha imediatamente um

pacote depois de ler o endereço de destino. O fragment-free, o switch armazena os primeiros 64 bytes do quadro antes de encaminhar.

1. O que são os buffers de memória do switch?

Um buffer armazena os dados na memória RAM devido ao tempo de acesso rápido. Essa tecnologia é ativada pelo sistema quando há uma diferença entre a taxa de recebimento e processamento de dados ou, em alguns casos, quando elas são variáveis, como em um serviço de streaming de vídeos.

1. **Quais são os tipos de buffer de memória do switch? Cite-os e diferencie.**

Memória por porta

- Os quadros são armazenados em filas vinculadas a portas específicas de entrada e saída.
- Um quadro é transmitido quando todos os quadros à frente dele forem transmitidos.

Memória compartilhada

- Todos os quadros são depositados em um buffer comum compartilhado por todas as portas no switch.
1. O que é a incompatibilidade de duplex?

É uma causa comum de problemas de desempenho com links de Ethernet. Ela ocorre quando uma porta no link opera em half-duplex, enquanto a outra porta opera em full-duplex.

1. O que é o recurso Auto-MDIX?

(Auto-MDIX) interface dependente automática. Quando habilitado usando o comando de configuração de interface mdix auto , o switch detecta o tipo de cabo conectado à porta e configura as interfaces de acordo.

1. **O que é e para que serve o protocolo ARP?**

Quando um dispositivo envia um quadro Ethernet, ele contém dois endereços:

- Endereço MAC destino e Endereço MAC origem

Para determinar o endereço MAC de destino, o dispositivo usa o ARP.

1. **Quais as funções básicas do protocolo ARP?**
- Resolução de endereços IPv4 em endereços MAC
- Manutenção de uma tabela de mapeamentos
1. **Explique passo a passo o funcionamento do protocolo ARP.**

Os dispositivos Ethernet consultam uma tabela do ARP em sua memória para localizar o endereço MAC mapeado para o endereço IPv4.

- Um dispositivo pesquisará em sua tabela ARP um endereço IPv4 destino correspondente a um endereço MAC.
- Se o endereço IPv4 destino do pacote estiver na mesma rede que o endereço IPv4 origem, o dispositivo pesquisará o endereço IPv4 destino na tabela ARP.
- Se o endereço IPv4 destino do pacote estiver em uma rede diferente do endereço IPv4 origem, o dispositivo pesquisará o endereço IPv4 do gateway padrão na tabela ARP.
1. **O que é a tabela ARP?**

O comando arp permite a criação, edição e exibição de mapeamentos de endereços físicos em endereços IPv4 conhecidos.

1. **O que é e como funciona a falsificação ARP?**

Invasores podem responder a solicitações fingindo ser prestadores de serviços. Um tipo de ataque de falsificação de ARP usado pelos invasores é responder a uma solicitação ARP para o gateway padrão.

Fornece serviços para permitir que dispositivos finais troquem dados pela rede.

1. **Quais são os processos utilizados pela camada de rede para oferecer conexão ponta-a-ponta?**

Endereçamento de dispositivos finais, encapsulamento, roteamento e desencapsulamento.

1. Quais os protocolos mais utilizados da camada de rede?
- Protocolo de Internet versão 4 (IPv4)
- Protocolo de Internet versão 6 (IPv6)
1. **Quais as características do protocolo IP?**

Foi desenvolvido como um protocolo com baixa sobrecarga – ele fornece apenas as funções necessárias para enviar um pacote da origem para um destino.

- Um pacote IP é enviado para o destino sem estabelecimento prévio de uma conexão
- O IP não foi projetado para rastrear e gerenciar o fluxo de pacotes.
1. **O que significa dizer que o protocolo IP é sem conexão?**

Nenhuma conexão dedicada de ponta a ponta é criada antes que os dados sejam enviados.

- Os remetentes não sabem se o destino está presente, acessível ou funcional antes de enviar pacotes, esse recurso contribui para a baixa sobrecarga de IP.
1. **O que significa dizer que o protocolo IP é um protocolo de entrega do melhor esforço ?**

O IP é considerado "não confiável" porque não garante que todos os pacotes enviados serão recebidos.

- Não confiável significa que o IP não tem a capacidade de gerenciar e recuperar pacotes não entregues, corrompidos ou fora de sequência.
- Se os pacotes estiverem ausentes ou fora da ordem correta no destino, os protocolos/serviços da camada superior deverão resolver esses problemas.
1. **Cite e fale sobre os principais campos do cabeçalho do protocolo IPv4.**

Um cabeçalho de pacote IPv4 consiste nos campos que contêm números binários. Esses números identificam várias configurações do pacote IP que são examinadas pelo processo da camada 3.

1. **Quais as limitações do protocolo IPv4 que tornaram o IPv6 necessário?**

Falta de endereços IP, expansão da tabela de roteamento da internet e falta de conectividade de ponta a ponta.

1. **Cite e fale sobre os principais campos do cabeçalho do protocolo IPv6.**

Versão – Contém um valor binário de 4 bits definido como 0110 que identifica que este é um pacote IPv6.

- Classe de Tráfego – O campo de 8 bits equivalente ao campo Serviços Diferenciados (DS) do IPv4.
- Rótulo de Fluxo – O campo de 20 bits sugere que todos os pacotes com o mesmo rótulo de fluxo recebam o mesmo tipo de tratamento pelos roteadores.
- Tamanho da Carga – O campo de 16 bits indica o tamanho da parte de dados ou da carga (payload) do pacote
- Próximo Cabeçalho – O campo de 8 bits é equivalente ao campo do protocolo IPv4. Ele indica o tipo de carga útil de dados que o pacote está carregando.
- Limite de Saltos – O campo de 8 bits substitui o campo Vida Útil (TTL) do IPv4. Esse valor é diminuído em 1 ao passar por cada roteador. Quando chega a zero, o pacote é descartado.
- Endereço Origem IPv6 – O campo de 128 bits que identifica o endereço IPv6 do host emissor.
- Endereço de destino IPv6 – O campo de 128 bits que identifica o endereço IPv6 do host destinatário.
1. **O que é o roteamento de pacotes?**

O roteamento seleciona os caminhos para que os pacotes do Protocolo de Internet (IP) viajem desde sua origem até seu destino. Essas decisões de roteamento na Internet são tomadas por hardwares de rede especializados chamados roteadores.

1. Quais são os três tipos de destino que um host pode enviar um pacote?
- Ele mesmo – um host pode fazer ping para ele para teste usando 127.0.0.1, conhecido como a interface de loopback.
- Host local – Este é um host na mesma rede do host emissor. Os hosts compartilham o mesmo endereço de rede.
- Host remoto – Este é um host em uma rede remota. Os hosts não compartilham o mesmo endereço de rede.
1. **O que é o gateway padrão?**

É o dispositivo de rede capaz de rotear o tráfego para outras redes.

1. **Como um host decide se deve enviar um pacote através do roteador padrão?**

O gateway padrão saberá onde enviar o pacote usando a tabela de roteamento.

1. **O que é a tabela de roteamento?**

A tabela de roteamento funciona como um mapa que guia os roteadores para acharem a rede de destino. Ela registra os caminhos que os pacotes devem seguir para chegar a todos os destinos pelos quais o roteador é responsável.
